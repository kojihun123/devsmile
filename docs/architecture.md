# Architecture

## 개요

Laravel 12 기반 쇼핑몰 포트폴리오 프로젝트.
모든 서비스는 Docker 컨테이너 위에서 실행되며, 개발/운영 환경 차이를 최소화하는 것을 목표로 한다.

## 컨테이너 구조

```
┌─────────────────────────────────────────────┐
│                   Browser                   │
└─────────────────────┬───────────────────────┘
                      │ :80
┌─────────────────────▼───────────────────────┐
│                    nginx                    │
│           정적 파일 서빙 / PHP 프록시           │
└─────────────────────┬───────────────────────┘
                      │ :9000 (FastCGI)
┌─────────────────────▼───────────────────────┐
│               app (php-fpm)                 │
│            PHP 8.4 / Laravel 12             │
└──────────┬──────────────────────┬───────────┘
           │                      │
┌──────────▼──────────┐  ┌───────▼───────────┐
│     mysql :3306     │  │    redis :6379     │
│    데이터베이스        │  │   캐시 / 세션       │
└─────────────────────┘  └───────────────────┘

┌─────────────────────────────────────────────┐
│               node :5173                    │
│           Vite 개발 서버 (HMR)               │
└─────────────────────────────────────────────┘
```

## 각 컨테이너 역할

| 컨테이너 | 이미지 | 역할 |
|---------|--------|------|
| nginx | nginx:alpine | 웹서버. 정적 파일은 직접 서빙, PHP 요청은 php-fpm으로 프록시 |
| app | php:8.4-fpm (커스텀) | Laravel 애플리케이션 실행 |
| node | node:22-alpine | Vite 개발 서버, HMR, Tailwind CSS 빌드 |
| mysql | mysql:8.0 | 메인 데이터베이스 |
| redis | redis:alpine | 세션 저장소, 캐시 |

## 요청 흐름

```
Browser → nginx(:80) → php-fpm(:9000) → Laravel → MySQL / Redis
                    ↘ 정적파일(.css/.js) 직접 응답
```

## 기술 스택 선택 이유

**Laravel Sail 미사용**
Sail은 내부적으로 Docker를 추상화해서 편리하지만, 실제 인프라 구조를 이해하기 어렵다.
nginx + php-fpm 구조를 직접 구성해서 실제 운영 환경과 동일한 구조를 유지한다.

**Redis 세션/캐시**
기본값인 database 드라이버 대신 Redis를 사용해 세션/캐시 성능을 높인다.

**Blade SSR**
React/Vue SPA 없이 Blade 템플릿으로 SSR을 유지한다.
Alpine.js로 필요한 인터랙션만 처리한다.

## 환경변수 주요 설정

```env
DB_HOST=mysql        # 컨테이너 이름으로 통신
REDIS_HOST=redis     # 컨테이너 이름으로 통신
SESSION_DRIVER=redis
CACHE_STORE=redis
```

Docker 네트워크 안에서는 컨테이너 이름이 호스트명으로 동작한다.
